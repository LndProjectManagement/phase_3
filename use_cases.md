# Описание проекта
По итогам Фазы 1 был выбран проект "GiftIdeas" - помощник в выборе и учете подарков для друзей и семьи: помнит, что дарили и что человек хотел.

# Use Cases

## 1. Регистрация нового пользователя
Предусловия:
  * Пользователь впервые открывает приложение.

Триггер:
  * Пользователь нажимает кнопку "Зарегистрироваться"

Основной сценарий:
  * Система предлагает ввести email(или номер телефона) и пароль.
  * Пользователь вводит email(или номер телефона) и пароль, соответствующий требованиям безопасности.
  * Система проверяет, что email(или номер телефона) еще не зарегистрирован.
  * Система отправляет шестизначный код подтверждения на указанный email(или номер телефона).
  * Пользователь вводит полученный код в соответствующее поле.
  * Система подтверждает код и предлагает ввести имя.
  * Пользователь вводит имя.
  * Система перенаправляет пользователя на главный экран.

Реузультат:
  * Создан аккаунт пользователя.

Альтернативный сценарий:
  * Если email уже занят, система сообщает: «Аккаунт с таким email уже существует. Попробуйте войти или восстановить пароль».
  * Если код введен неверно, система отображает ошибку: «Неверный код подтверждения».

## 2. Добавление получателя подарков
Предусловия: 
  * Пользователь авторизован.

Триггер: 
  * Пользователь нажимает кнопку "Добавить близкого" на главном экране или в соответствующем разделе "Близкие".

Основной сценарий:
  * Приложение открывает форму с полями: Имя, тип отношения (друг, семья, коллега и т.д.), дата рождения, фото, интересы (теги), заметки.
  * Пользователь заполняет обязательные поля (Имя) и опциональные (дата рождения, интересы).
  * Пользователь нажимает "Сохранить".
  * Система валидирует данные (например, корректность даты).
  * Система создает карточку близкого и отображает ее в общем списке.

Результат: 
  * В системе создан новый профиль близкого человека.

Альтернативный сценарий:
  * Некорректная дата: Система показывает ошибку "Укажите корректную дату".
  * Пользователь возвращается к шагу 2 основного сценария.

## 3. Добавление идеи в вишлист
Предусловия: 
  * Пользователь авторизован.
  * В системе существует как минимум один профиль близкого человека.

Триггер: 
  * Пользователь находится в карточке близкого, в разделе "Вишлист", и нажимает "Добавить идею".

Основной сценарий:
  * Система открывает форму: название подарка, описание, ссылка на товар, категория, приоритет, примерная стоимость.
  * Пользователь заполняет название и категорию.
  * Пользователь нажимает "Сохранить".
  * Система добавляет эту идею в вишлист выбранного близкого.

Результат: 
  * В вишлисте близкого человека появилась новая идея для подарка.

Альтернативный сценарий:
  * Добавление по ссылке: Пользователь вставляет ссылку на товар в интернете. Система пытается парсить заголовок, описание и картинку, чтобы автоматически заполнить форму.

## 4. Запись подаренного подарка
Предусловия: 
  * Пользователь авторизован.
  * В системе существует как минимум один профиль близкого человека.

Триггер: 
  * Пользователь заходит в карточку близкого и нажимает "Добавить в историю".

Основной сценарий:
  * Система открывает форму: выбор подарка из вишлиста (опционально), название подарка, описание, дата дарения, категория, фото, реакция получателя.
  * Пользователь вводит название подарка и выбирает дату дарения (по умолчанию - сегодня).
  * Пользователь нажимает "Сохранить".
  * Система добавляет запись в раздел "История подарков" профиля близкого.

Результат: 
  * В истории подарков близкого человека появилась новая запись. Система учитывает этот подарок для будущих рекомендаций.

Альтернативный сценарий:
  * Подарок был в вишлисте: Пользователь выбирает подарок из выпадающего списка "Из вишлиста".
  * Поля "Название" и "Категория" заполняются автоматически.
  * Подарок помечается как "Подаренный" и перемещается в историю.

## 5. Просмотр рекомендаций
Предусловия: 
  * Пользователь авторизован.
  * В системе есть близкий с заполненными интересами или историей подарков.

Триггер: 
  * Пользователь заходит в карточку близкого и переходит в раздел "Рекомендации".

Основной сценарий:
  * Система анализирует интересы (теги) и историю подарков близкого.
  * Система генерирует список идей, исключая уже подаренные подарки.
  * Система отображает список рекомендаций с возможностью добавить любую идею в вишлист одной кнопкой.

Результат: 
  * Пользователь видит персонализированный список идей для подарка.

Альтернативный сценарий:
  * Недостаточно данных: Если у близкого нет интересов и истории, система показывает сообщение: "Добавьте интересы и историю подарков, чтобы получать точные рекомендации", и предлагает общие популярные подарки.

## 6. Просмотр главной ленты подарков
Предусловия: 
  * Пользователь авторизован.

Триггер: 
  * Пользователь открывает главный экран приложения.

Основной сценарий:
  * Система отображает ленту с карточками подарков из предзагруженного каталога.
  * Над лентой система показывает два основных фильтра: "Категории" (например, Книги, Электроника, Для дома) и "Для кого" (список близких людей).
  * Пользователь просматривает общий каталог.
  * Пользователь может нажать на карточку подарка, чтобы увидеть детали и добавить его близкому.

Результат: 
  * Пользователь имеет доступ к обширной базе идей для подарков.

Альтернативный сценарий:
  * Пользователь применяет фильтр "Категории": Лента мгновенно обновляется, показывая только подарки из выбранной категории.
  * Пользователь применяет фильтр "Для кого": Система анализирует интересы выбранного близкого и фильтрует ленту, показывая подарки, релевантные его интересам.

## 7. Поиск по подаркам и людям
Предусловия: 
  * Пользователь авторизован.

Триггер: 
  * Пользователь вводит текст в строку поиска на главном экране.

Основной сценарий:
  * Система в реальном времени ищет совпадения по именам близких, названиям подарков, категориям и интересам.
  * Система отображает результаты, сгруппированные по типам: "Люди", "Подарки", "Интересы".
  * Пользователь выбирает нужный результат из списка.

Результат: 
  * Пользователь переходит к найденной карточке человека или подарка.

Альтернативный сценарий:
  * Совпадений не найдено: Система показывает сообщение "По вашему запросу ничего не найдено".

## 8. Аналитика потраченных средств
Предусловия: 
  * Пользователь авторизован.
  * В истории подарков есть записи с указанной стоимостью.

Триггер: 
  * Пользователь переходит в раздел "Аналитика бюджета".

Основной сценарий:
  * Система показывает график/диаграмму общих затрат по месяцам/годам.
  * Система показывает распределение затрат по людям или по категориям.
  * Пользователь может выбрать период для анализа.

Результат: 
  * Пользователь получает наглядное представление о своих расходах на подарки.

Альтернативный сценарий:
  * Нет данных о стоимости: Система показывает сообщение "Для анализа добавьте примерную стоимость в историю подарков".

## 9. Создание праздничного списка
Предусловия: 
  * Пользователь авторизован.
  * В системе есть несколько близких.

Триггер: 
  * Пользователь нажимает "Создать новый список" в разделе "События".

Основной сценарий:
  * Система запрашивает название списка.
  * Пользователь выбирает близких из общего списка для добавления в этот событийный список.
  * Для каждого человека в списке можно добавить идею из вишлиста или создать новую.
  * Пользователь нажимает кнопку "Сохранить".
  * Система сохраняет список.

Результат: 
  * Создан тематический список для удобного планирования подарков к событию.

Альтернативный сценарий:
  * Пользователь удаляет человека из списка: Человек удаляется только из событийного списка, его основной профиль остается нетронутым.

## 10. Собственный вишлист
Предусловия: 
  * Пользователь авторизован.

Триггер: 
  * Пользователь переходит в раздел "Мой аккаунт -> Мой вишлист".

Основной сценарий:
  * Система отображает интерфейс, аналогичный вишлисту близкого.
  * Пользователь добавляет, редактирует или удаляет желаемые подарки.
  * Пользователь может нажать "Поделиться вишлистом", чтобы отправить ссылку родным.

Результат: 
  * Пользователь ведет персональный список желаний.

## 11. Удаление аккаунта
Предусловия: 
  * Пользователь авторизован.

Триггер: 
  * Пользователь заходит в "Мой аккаунт -> Настройки -> Удалить аккаунт".

Основной сценарий:
  * Система показывает предупреждение о необратимости действия и запрашивает подтверждение пароля.
  * Пользователь вводит свой пароль и нажимает "Удалить аккаунт".
  * Система проверяет пароль.
  * Система удаляет все данные пользователя из базы данных.
  * Система разлогинивает пользователя и перенаправляет на экран приветствия.

Результат: 
  * Аккаунт и все данные безвозвратно удалены.

Альтернативный сценарий:
  * Неверный пароль: Система показывает ошибку "Неверный пароль".
  * Пользователь возвращается к шагу 2.
